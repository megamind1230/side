* DotaDraftAdvisor
:PROPERTIES:
:UPDATED: 2026-06-17 (Phases 4-6 complete)
:END:

Pick your draft. See the score. Counter-pick like a pro.

A console-based Dota 2 draft advisor that analyzes real hero matchup
data from the OpenDota API and recommends the best 5th pick. Built to
demonstrate async/await, Task Parallel Library, ThreadPool, BackgroundWorker,
race conditions, and deadlocks — all with live data.

** Features
- Manual draft analysis: input 4 ally + 5 enemy picks, get ranked 5th pick recommendations
- Hero aliases: type "pa", "lc", "sf" — resolves to full hero names
- Real data: uses OpenDota API win/loss matchup stats (not synthetic)
- Role analysis: identifies what your team lacks (initiation, magic damage, wave clear, etc.)
- Race condition demo: see what happens when Parallel.ForEach writes to a plain Dictionary — then watch ConcurrentDictionary fix it
- Deadlock demo: two threads acquire locks in opposite order and freeze — then fixed with consistent lock ordering
- Live draft watcher: BackgroundWorker polls OpenDota /api/live every 30s, detects matches in picking phase, and recommends counter picks in real time
- Rich console UI with tables, colors, and progress bars (Spectre.Console)
- Full Serilog logging to ~/magnus/DotaDraftAdvisor/logs/

** Async Concepts Demonstrated

| Concept              | Location                      | What happens                                                        |
|----------------------+-------------------------------+---------------------------------------------------------------------|
| async/await          | OpenDotaService               | 9 concurrent API calls via Task.WhenAll — no threads blocked on I/O |
| Task.WhenAll         | Program.cs / DraftScorer      | Fetch all matchups in parallel, wait for all to complete            |
| Parallel.ForEach     | DraftScorer                   | Score 115 heroes across CPU cores simultaneously                    |
| PLINQ                | DraftScorer                   | .AsParallel().OrderByDescending() for sorting results               |
| ConcurrentDictionary | DraftScorer (fixed)           | Thread-safe scoring dictionary — no torn reads/writes               |
| Interlocked          | DraftScorer (fixed)           | Lock-free increment for aggregate counters                          |
| ThreadPool           | DraftScorer deep-dive         | QueueUserWorkItem for top-5 hero analysis tasks                     |
| BackgroundWorker     | LiveDraftWatcher              | Polling loop + progress reporting + cancellation via CancellationToken |
| lock                 | DeadlockSection               | Mutual exclusion on AllyPool / EnemyPool resources                   |
| Monitor.TryEnter     | DeadlockSection (broken)      | Deadlock detection with 3s timeout                                  |
| Lock ordering        | DeadlockSection (fixed)       | Consistent acquisition order eliminates circular wait                |
| Barrier              | DeadlockSection (broken)      | Sync threads to guaranteed deadlock interleave point                 |

** Scoring Algorithm

Final score = (winRateScore × 0.55) + (roleCoverage × 0.25) + (laneFit × 0.20)

- *Win rate score*: Average win rate across all 5 enemy matchups.
  Bayesian shrinkage pulls low-sample heroes toward 50%.
- *Role coverage*: +15 pts per missing role filled (initiator, magic
  damage, physical damage, tank, wave clear, roamer).
- *Lane fit*: +10 pts if hero fits an open lane based on their
  most common role.

** API Rate Limits

OpenDota API allows ~60 req/min for anonymous users. The app fetches
9 matchup sets (5 enemy + 4 ally) per analysis, well within limits.
Matchup data is cached in memory for the session — re-analyzing with
the same heroes uses cached data.

** Implementation Notes
- All API calls are async — no thread blocking during HTTP requests
- Race condition demo deliberately introduces a bug for educational purposes
- Live draft watcher uses CancellationToken for clean shutdown
- Hero alias list covers ~70 common shorthands — does not include all heroes
- OpenDota API returns 124+ heroes; the app excludes the 9 already picked
- Spectre.Console AnsiConsole is thread-safe for live display updates
- Deadlock demo uses Barrier for guaranteed deadlock interleave (not sleep-dependent)
