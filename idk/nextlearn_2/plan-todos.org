* DONE AvaloniaEdit integration (replaces Markdown.Avalonia)

** Why
Markdown.Avalonia hides `#`/`##`/`*`/`**` heading markers. User wants
visible source markers in the content area.

** Resolution
- Replaced Markdown.Avalonia + SyntaxHigh with ~AvaloniaEdit~ +
  ~AvaloniaEdit.TextMate~ + ~TextMateSharp.Grammars~ (v11.1.0 for
  Avalonia 11.3.12 compatibility)
- ~TextEditor~ configured read-only, transparent background, word wrap
- TextMate syntax highlighting with DarkPlus theme, grammar switches
  between ~.md~ and ~.org~ by extension
- ~LearningViewModel.RenderedContent~ prepends the heading prefix
  (~~## ~~/~~~** ~~) + page title to ~TextContent~ so the heading
  line appears as syntax-highlighted source in the editor

** Files changed
- ~NextLearn.Desktop.csproj~ — added 2 package references
- ~ViewModels/LearningViewModel.cs~ — +RenderedContent, IsOrgFile, HasSection
- ~Views/MainWindow.axaml~ — teal content area switched from TextBlock
  to ~ae:TextEditor~; top bar split into DeckTitle + breadcrumb rows
- ~Views/MainWindow.axaml.cs~ — +TextMate Installation, SetupEditor(),
  UpdateEditor() wired via PropertyChanged

** Bug fixed
- Teal box showed no content on first load because ~OnLoaded~
  subscribed to ~PropertyChanged~ after the initial ~RenderedContent~
  value was already set. Fixed by calling ~UpdateEditor()~ at the end
  of ~OnLoaded~.

* DONE P2: Stabilize deck identity

** Why
File-based decks got ~Guid.NewGuid()~ every session, so UserProgress
was orphaned after restart.

** Resolution
Created ~Services/DeckFileIdentity.cs~ which generates a deterministic
GUID from the SHA256 hash of the full file path. Same path → same GUID
every session. Progress survives restarts.
