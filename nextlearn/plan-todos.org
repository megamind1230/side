* DONE Anki Flashcard Generation — Google Gemini

** What
New "Anki Flashcards" sidebar panel with two dedicated buttons: [Basic] and
[Cloze]. Each generates one card type via a separate focused prompt. Preview
raw output → accept saves as ~.basic.txt~ / ~.cloze.txt~ under
~$HOME/magnus/nextlearn/flashcards~ for direct Anki import.

** Key decisions
- Two independent buttons instead of one mixed request: user controls type,
  no partial-merge complexity
- Separate ~.basic.txt~ / ~.cloze.txt~ files so Anki import maps each file to
  correct note type (mixed types in one import file not supported by Anki)
- Cloze format is single-line (no tabs): ~sentence with {{c1::blank}}~ —
  Anki's Cloze note type has only one text field
- ~_generationId~ counter discards stale responses if user clicks a new
  generation before previous one completes
- Each prompt says "Cover EVERY section — do not skip any part" to prevent
  lazy/partial generation
- Flashcards path defaults to ~$HOME/magnus/nextlearn/flashcards~ (same
  pattern as DecksPath: configurable in Settings with Browse button)
- File naming: ~<deck-filename>.basic.txt~ and ~<deck-filename>.cloze.txt~

** Basic prompt
- File: ~Services/FlashcardService.cs~ — ~BuildBasicPrompt~
- Requests TSV (tab-separated) format: ~term TAB definition/answer~
- One card per line, no markdown, no code fences

** Cloze prompt
- File: ~Services/FlashcardService.cs~ — ~BuildClozePrompt~
- Requests single-line format (no tabs): ~sentence with {{c1::blank}}~
- One blank per card, no markdown, no code fences
