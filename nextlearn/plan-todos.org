** Post-merge fix: zoom centering with ScrollChanged (B26)
Replaced one-shot ~LayoutUpdated~ with persistent ~ScrollViewer.ScrollChanged~ + ~_pendingRecenter~ flag.
~LayoutUpdated~ fires for ANY layout change (slider redraw) before image size change is processed,
reading stale ~Extent~/~Viewport~ values. ~ScrollChanged~ fires after extent changes, gated by
~ExtentDelta != default~. Added center vs preserve mode.

Initial open centers image: ~offset = (ext − vp)/2~.

Zoom preserve mode originally used ~offset = oldOffset + (newExt − oldExt)/2~ — viewport-agnostic formula that
only works when image is perfectly centered (~oldOffset = (ext − vp)/2~). For panned images, the formula
is wrong (e.g., Y offset jumps 75px on 1.5× zoom). Fixed to:
~newOffset = (oldOffset + vp/2) × zoomRatio − vp/2~, which correctly factors in viewport.

Also fixed async initial-center override: ~CurrentImageBitmap~ was grouped with ~ZoomLevel~/~RotationAngle~
using ~centerInViewport: false~, overriding the true intended by the ~IsImageOverlayOpen~ handler (which
bailed because ~Source~ was still null during async bitmap load). Separated into its own handler
with ~centerInViewport: true~.

* DONE Wayland/GLFW startup crash fix

** What
On Hyprland (Wayland), NativeWebView's embedded GLFW tried to initialize
the Wayland backend, got ~wl_display_connect("")~ failure, emitted 10+
~[glfw error 65544]: Wayland: Failed to connect to display~ messages.
The app still ran but with terminal spam.

** Root cause
~Environment.SetEnvironmentVariable("WAYLAND_DISPLAY", "")~ calls
~setenv("WAYLAND_DISPLAY", "")~ → GLFW's ~getenv("WAYLAND_DISPLAY")~
returns a non-NULL pointer (empty string), so GLFW thinks Wayland IS
configured and tries to connect. Connect fails because display name is "".

** Fix
~Environment.SetEnvironmentVariable("WAYLAND_DISPLAY", null)~ calls
~unsetenv("WAYLAND_DISPLAY")~ → ~getenv()~ returns NULL → GLFW skips
the Wayland backend entirely and falls through to X11.

Also forces X11 backend via ~UseSkia().UseX11()~ when
~XDG_SESSION_TYPE=wayland~.

** Files changed
- ~Program.cs~ — ~SetEnvironmentVariable("WAYLAND_DISPLAY", null)~ +
  ~UseSkia().UseX11()~ in Wayland branch
